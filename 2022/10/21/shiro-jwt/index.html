SpringBoot整合Shiro+JWT 实现认证授权

介绍

SpringBoot：不说了，大家都懂，

Shiro：一个安全框架，可以实现用户的认证和授权，相比 SpringSecurity 要简短的多。

Jwt：客户端和服务端之间进行验证的方式，取代了 session 的不安全性

4、把令牌对象封装成认证对象

Shiro框架认证需要用到认证对象，把令牌字符串进行封装。返回 AuthenticationToken 类型的对象

5、实现认证，授权功能

6、利用缓存刷新Token过期时间

Token 过期代表着用户需要重新登录，假设用户正在使用时，Token过期，不就会影响用户的体验。为此，设计方案利用Redis缓存进行刷新

Token缓存是把 Token 缓存到 Redis，设置缓存的 Token 过期时间为正常 Token 的1倍，然后根据用户的使用情况进行刷新缓存。

Token 过期，缓存没有过期

说明用户最近还正常使用，不应该让用户重新登录。将根据获取到的 Token 信息返回新的 Token 给用户，同时刷新 Redis 里的过期时间。

Token 过期，缓存也过期

说明用户较长时间没有使用，应该让用户重新登录。创建新的 Token 返回用户，添加缓存过期时间。

令牌刷新流程

7、创建本地缓存保存Token

8、创建过滤器进行判断

在过滤器中需要读取本地缓存 ThreadLocal 中的数据，需要将过滤器类设置为多例对象 使用注解 @Scope("prototype")

9、创建 ShiroConfig 类

ShiroConfig 是把 OAuth2Filter 和 OAuthRealm 配置到 Shiro 框架，这样 Shiro + JWT 才会生效

10、利用AOP把更新令牌

之间把更新后的令牌存放到本地缓存中 ThreadLocalToken 里面的 ThreadLocal 中，利用 AOP 把缓存中数据返回给用户。

11、精简异常

默认异常返回信息过多，容易暴露个人信息，需要进行简化
