Docker

Docker 官网：http://www.docker.com

Docker 仓库官网：https://hub.docker.com/

​ 传统虚拟机是虚拟出一套硬件后，在其运行一个完整的操作系统，在操作系统上运行进行。

​ Docker 是借助于Linux 平台，轻量级的虚拟机产品。使用 Docker 不会为每个虚拟化实例创建完整的虚拟硬件环境，只实例少量的硬件环境（网卡等），这些虚拟化环境是共享 Linux系统的 CPU，内存，硬盘等资源。

​ Docker 通过创建容器来实现多个运行环境，每个容器之间是相互隔离的。有自己的文件系统，容器之间的进行不会相互影响，能够区分计算资源。

Docker 组成

镜像： 是一个只读模板，镜像可以用来创建容器，一个镜像可以创建很多容器。（可以理解为 VMware 软件，可以加载多个虚拟机系统）

容器： 镜像创建出来的运行实例，可以将各种环境安装到不同容器，实现不同功能。（可以理解为 Vmware 下的多种操作系统，CentOS，Ubuntu，RedHat）

仓库： 是集中存放镜像文件的场所，最大的公开仓库为：https://hub.docker.com/（可以理解为 Maven 存放 jar 包的仓库，GitHub 存放 git 项目的仓库）

Docker vs VM

先说结论：Dokcer 比 传统虚拟机 快

Docker 有着比虚拟机更少的抽象层： 由于 docker 不需要 虚拟机实现硬件资源虚拟化，运行在 docker 容器上的程序直接使用的都是实际物理机的硬件资源，因此在 CPU、内存利用率上 docker 将会在效率上有着明显的优势。

Docker 利用的是宿主机的内核，不需要加载操作系统 OS 内核： 当新建一个容器时，docker 不需要和虚拟机一样重新加载一个操作系统内核，直接利用宿主机的操作系统，省略了返回过程，只需要几秒钟的时间。而新建一个虚拟机需要重新加载 OS，需要几分钟的时间

Docker容器 虚拟机
操作系统 与宿主机共享 OS 宿主机OS 上运行虚拟机 OS
存储大小 镜像小，便于存储和传输 镜像庞大
运行性能 几乎无额外性能损失 操作系统额外的 CPU，内存消耗
移植性 轻便、灵活，适应于Linux 笨重，与虚拟化技术耦合度高
硬件亲和性 面向软件开发者 面向硬件运维者
部署速度 快速，秒级 较慢，分级

容器卷

容器卷就是用来本地存储容器的内存，保存容器备份文件，将容器数据保存到主机磁盘

容器添加映射后，宿主机和容器之间的目录就进行同步共享，拥有的文件一致

默认情况下，容器进行挂载后与宿主机之间可以实现 读写 操作。

经过设置，也可以使容器被限制，只能读不能写。

容器卷的继承和共享

容器 1 完成和宿主机的映射后，可以和宿主机进行文件共享

容器 2 继承容器 1 的卷规则后，仍然会和宿主机进行文件共享，不管容器 1 的状态

即：容器 2 继承的是容器 1 的规则，继承了和宿主机共享的映射，容器 2 不与容器 1 之间有直接关联

常用安装

Tomcat

默认情况下

从官网仓库下拉取到是最新的，试连接发现 404，需要进行配置

进入 Tomcat 文件目录下： / usr / local / tomcat 查看文件内容，发现存在 webapps ，webapps.dist。将 webapps 删除，将 webapps.dist 改名为 webapps

启动即可。

Tomcat 8 版本

docker run -d -p 8080:8080 –name mytomcat8 billygoo/tomcat8-jdk8

直接下载 Tomcat 8 版本，不需要进行配置。

MySql

下载 MySql 数据库 使用 5.7 版本 docker pull mysql:5.7

由于数据需要进行备份，采用容器卷方式保存，并且默认下载的数据库不支持中文，需要进行配置。

方便理解方式

添加配置文件（目录在启动时设置的 /echo/mysql/conf ）

重新查看数据库

因为数据库已经通过数据卷进行备份，所以即使删除 MySql 容器，数据仍然保存到了宿主机上，再次启动即会加载备份的数据文件，达到数据恢复

Redis

下载 Redis 版本 使用 6.0.8 版本， docker pull redis:6.0.8

由于数据需要进行备份，采用容器卷方式保存，并且配置 redis 的配置文件，保存到宿主机，方便更改 redis 配置

方便理解方式

启动命令中采用 /ect/redis/redis.conf 启动，映射 宿主机的 /echo/redis/redis.conf

将宿主机的 redis.conf 进行配置

本地启动

​ 可以将容器设置成 host 模式，就是不让 docker 为容器创建虚拟网卡，使用宿主机的网卡，这样 多个 docker 容器 之间就都使用的是 宿主机的网卡。

A 容器中的 localhost 代表的就是 宿主机的网卡，B 容器中的 localhost 也代表的是 宿主机的网卡，两个容器相互访问。URL 地址写 localhost 即可。
